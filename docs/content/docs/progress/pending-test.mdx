# 待测试

- 视频生成轮询已支持创建任务时保存 `video_id`，后续优先用 `GET /agnesapi?video_id=...` 查询，并在完成响应中优先读取直返视频 URL，缺少直返 URL 时再回退 `/content`；需要验证 Agnes 等返回 `video_id` 的渠道排队状态和最终视频获取正常，旧的仅返回 `task_id` 渠道仍可生成。
- 顶部和移动端导航已恢复视频创作台入口，画布、生图工作台和视频创作台生成按钮的算力点显示改为读取系统模型价格配置；需要验证 `/video` 可从导航进入，并且图片、视频、文本、音频生成按钮显示的算力点与管理后台模型价格一致。
- 提示词接口已从 Next.js 本地路由切回 `/api` 代理到 Go 后端，Docker 容器内 GitHub 同步请求增加本机代理环境变量；需要验证用户前端提示词页能从后端 DB 正常加载提示词，管理后台提示词同步在服务器环境可用。
- 私有模型渠道已从渠道级模型前缀改为每个真实模型单独配置前台显示名称；需要验证管理后台可保存中文显示名，用户前端模型下拉和算力点价格使用显示名，生成请求能由后端还原为真实上游模型名。
- 在线 `Agent` 工具卡片详情不再保存和展示完整上下文消息，只展示当前工具调用和执行结果；需要验证详情内容体积不会随对话历史膨胀。
- 文本生成和在线 `Agent` 的 Responses API 请求已改为流式输出；需要验证生成文本、工具确认前后的对话回复会实时刷新。
- 在线 `Agent` 自动执行工具时会显示已完成工具卡片，并且从生成配置节点触发生成时会使用配置节点的提示词；需要验证关闭工具确认后连接节点并生图时请求不再发送空 `prompt`。
- 网站端在线 `Agent` 工具已对齐本地 Agent 的通用画布工具，支持创建节点、创建生成流程、直接触发文本/图片/视频/音频生成和 `canvas_apply_ops`；需要验证只有文本节点时也能通过 `canvas_generate_image` 创建生图流程并执行。
- 网站端在线 `Agent` 新建生成配置节点时会继承当前页面的默认生成设置，包括生图模型、尺寸和数量；需要验证 AI 创建生图流程时使用的模型和手动添加生成配置节点一致。
- 本地版本更新系统已实现：后端新增 `releases` 表和 API，支持管理员手动维护和 AI 自动生成版本记录；用户前端重新开放版本更新入口，展示我们自己的版本更新记录，管理后台保留上游 GitHub 版本更新展示。需要验证用户端版本弹窗正确展示后端版本记录、管理后台版本记录页面支持新建/编辑/删除/AI 生成、管理后台头部仍然显示上游版本更新。
